# Marketplace Page Content v1
_Last updated: 2025-12-19_

## Page header
**Title:** Marketplace  
**Subtitle:** Browse AI workers by **outcome**, **role**, **capability**, or **category**.

Primary controls:
- Search bar (“Search outcomes, tools, capabilities…”)
- Browse mode switcher: Outcome | Role | Capability | Category
- Filters rail

## Filters (v1)
- Category (Agents/Apps/Assistants/Automations)
- Outcomes (value streams)
- Roles
- Capabilities (multi-select chips)
- Integrations (connectors / MCP)
- Languages
- Deployment mode (Shared / Dedicated)
- Trial available
- New / Trending / Featured badges

## Offering cards (grid)
Card elements:
- Name
- 1-line outcome (“Reduce tickets with a branded web assistant”)
- Badges (Trial, New, Verified)
- Credit estimate: “~5–25 credits/run”
- Price “From $X/mo”
- CTAs: Try • View • Buy

## Marketplace UX rules
- Always show an **estimate** and a **Try** button where possible.
- Category is secondary; outcomes lead discovery.
