# GTM UX Specification: Agent.ai + MuleRun Pattern

**Date:** December 16, 2025
**Status:** Implementation Specification
**Priority:** P0 - GTM Critical
**Target:** Launch-ready marketplace UX

---

## Executive Summary

For GTM, adopt a **simplified, consumer-friendly marketplace UX** inspired by:
- **Agent.ai** - Professional network for AI agents with social proof
- **MuleRun** - Pay-per-run agent store with credit system

This approach prioritizes:
1. **Instant access** (no approval workflows)
2. **Credit-based pricing** (usage-based, transparent)
3. **Run-first UX** (try immediately, pay later)
4. **Social proof** (executions, ratings, reviews)

Post-GTM, upgrade to enterprise patterns (Salesforce Agentforce, Kore.ai).

class CreditPackage(models.Model):

    credits = models.IntegerField()  # 100, 1000, 6000
    price = models.DecimalField()  # 0, 10, 50
    bonus_credits = models.IntegerField(default=0)  # 0, 0, 1200
    stripe_price_id = models.CharField()

class CreditBalance(models.Model):
    """User/Org credit balance"""
    customer = models.OneToOneField(Customer)
    balance = models.IntegerField(default=0)
    free_credits_claimed = models.BooleanField(default=False)

class CreditTransaction(models.Model):
    """Credit purchase/spend history"""
    customer = models.ForeignKey(Customer)
    amount = models.IntegerField()  # Positive = purchase, Negative = spend
    transaction_type = models.CharField()  # "purchase", "spend", "bonus", "refund"
    reference_type = models.CharField()  # "agent_run", "package_purchase"
    reference_id = models.UUIDField()
    created = models.DateTimeField(auto_now_add=True)
